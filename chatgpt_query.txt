Сгенерировать csv длиной в 100000, в котором будет следующие поля:
1. Временная метка действия с точностью до секунды (тип данных timestamp), колонка с названием timestamp
2. ID клиента (тип данных int), колонка с названием client_id
3. Вид активности (активности из списка: клик по рекламе, клик на переход с информации на ввод телефона, ввод телефона, ввод смс, ввод паспорта, ввод почты, ввод фио, сабмит, приход в отделение, взял займ) (тип данных string), колонка с названием activity
4. Фичи - колонка features (тип данных json). Если вид активности клик по рекламе или  клик на переход с информации на ввод телефона или сабмит или приход в отделение или взял займ, то фичи должны быть null. Если вид активности ввод телефона, то фичи должны быть json с ключом value и значением номер телефона (из 10 цифр), с ключом num_of_keypress и значением в 70% случаев равным 10, в 10% случаев равным 2, в 7% случаев равным 0 и в остальных случаях должен быть четным между 12 и 26, с ключом num_of_backspace и значением равным 0 если num_of_keypress <= 10, иначе int((num_of_keypress - 10) / 2). Если вид активности ввод смс, то фичи должны быть json с ключом value и значением код из смс (из 4 цифр), с ключом num_of_keypress и значением в 60% случаев равным 4, в 20% случаев равным 0 и в остальных случаях должен быть четным между 6 и 8, с ключом num_of_backspace и значением равным 0 если num_of_keypress <= 4, иначе int((num_of_keypress - 4) / 2). Если вид активности ввод паспорта, то фичи должны быть json с ключом value и значением номер паспорта (из 10 цифр), с ключом num_of_keypress и значением  в 75% случаев равным 10, в 10% случаев должен быть равен 2, в 5% случаев должен быть равен 0 и в остальных случаях должен быть четным между 10 и 20, с ключом num_of_backspace и значением равным 0 если num_of_keypress <= 10, иначе int((num_of_keypress - 10) / 2). Если вид активности ввод почты, то фичи должны быть json с ключом value и значением электронной почты, с ключом num_of_keypress и значением в 65% случаев равным длине значения value, в 10% случаев должен быть равен 2, в 5% случаев должен быть равен 0 и в остальных случаях должен быть четным между длиной value и длиной value + 10, с ключом num_of_backspace и значением равным 0 если num_of_keypres <= длина value, иначе int((num_of_keypress - длина value) / 2). Если вид активности ввод фио, то фичи должны быть json с ключом value и значением фамилии имени и отчества, с ключом num_of_keypress и значением в 80% случаев должен быть равен длине значения value, в 10% случаев должен быть равен 2, в 3% случаев должен быть равен 0 и в остальных случаях должен быть четным между длиной value и длиной value + 10 с ключом num_of_backspace и значением равным 0 если num_of_keypres <= длина value, иначе int((num_of_keypress - длина value) / 2).


Самое важное, чтобы активности для каждого клиента шли в порядке указанном выше (в порядке возрастания), 10% клиентов должны по порядку сделать все действия, 30% клиентов должны сделать только первую активность,  20% клиентов должна сделать сначала первую активность, потом вторую, 5% должны сделать сначала первую активность, потом вторую, потом третью, 5% должны сделать сначала первую активность, потом вторую, потом третью, потом четвертую, 5% должны сделать последовательно первые пять активностей по порядку,5% должны сделать последовательно первые шесть активностей по порядку, 5% должны сделать последовательно первые семь активностей по порядку, 5% должны сделать последовательно первые восемь активностей по порядку, 10% должны сделать первые девять активностей по порядку.  Разница между первым активностями должна быть между 0 и 500 секундами, разница между сабмитом и приходом в отделение должна быть между 0 и 5 дней, разница между приходом в отделение и займом должна быть от 15 до 60 минут. Каждый клиент должен последовательно во времени (по timestamp) делать активности.

Почты, номера телефонов, коды должны быть различными для разных id клиентов и одинаковыми для одинаковых значений id клиента.

Фамилии, имена и отчества должны быть английскими и различными для разных id клиентов и одинаковыми для одинаковых значений id клиента,  используй как минимум 20 различных имен, 50 различных фамилий и 25 различных отчеств.
